# Elara

A Discreet, Privacy-First Incident Logging Platform

"A Place to be Heard"

## Overview

Elara is a privacy-first web application designed to help individuals safely document sensitive personal incidents in environments where visibility and security are critical concerns.

This project was developed as part of the **Anthropic-sponsored hackathon hosted by Vanderbilt’s Claude Builder’s Club**, with a focus on building responsible, safety-aware technology using modern AI tools.

Elara operates behind a **disguised interface**, appearing as a normal, everyday website such as a grocery store, encyclopedia page, legal document portal, or fitness tracker. Beneath this surface, it provides a hidden, structured system for logging incidents, organizing evidence, and generating formal reports.

The goal of Elara is not to replace professional services, but to reduce the friction of documentation and help users maintain accurate, organized records in situations where doing so may otherwise be difficult or unsafe.

---

## Key Features

### Discreet Access Through Hidden Inputs

Elara does not expose any visible entry point into its secure system. Instead, access is triggered through carefully placed, invisible interactions embedded within each theme. These inputs are designed to blend seamlessly into the surrounding interface, making them undetectable during casual observation.

Only one access point exists per theme, reducing the risk of accidental discovery.

---

### Structured Incident Logging and Report Generation

Users can log incidents with structured data, including:

* Date and time
* Description
* Location
* Severity
* Additional contextual details

All entries are automatically compiled into a **formal, police-style report format**, organizing incidents into a clear, chronological, and legally useful structure. Missing fields are handled gracefully, ensuring consistency across reports.

---

### Voice-to-Text Input

Elara includes a voice recording feature that allows users to dictate entries instead of typing. Speech is converted to text in real time and appended to existing content, enabling quick, natural input in time-sensitive situations.

---

### Image Uploading for Evidence

Users can attach images to individual entries to support documentation, such as photos or screenshots. These are stored alongside incident logs and included in generated reports as part of the evidence record.

---

### Automatic Session Timeout

To protect user safety, Elara automatically exits after a period of inactivity. If no interaction is detected, the session is closed and the user is redirected to a neutral external site. This helps reduce the risk of exposure if the device is left unattended.

---

### Intelligent Alerts and Safety Signals

Elara includes built-in detection for high-risk language patterns within entries, including:

* Threat-related indicators (e.g., weapons)
* Self-harm or crisis-related language

When detected, the system provides **discreet, non-intrusive alerts** that encourage the user to seek support or take immediate action, without drawing unnecessary attention on screen.

---

### Location-Aware Support

When generating reports or accessing support features, Elara can optionally use the user’s location to:

* Suggest nearby police stations
* Surface relevant local resources
* Provide context-aware support options

All actions remain user-controlled, with no automatic reporting or external communication.

---

## Design Philosophy

Elara is built around three core principles:

**Discretion**
The interface must remain indistinguishable from normal browsing activity.

**Control**
All actions, especially those involving reporting or external communication, require explicit user confirmation.

**Safety Awareness**
The system should support users without introducing additional risk or unnecessary exposure.

---

## Notes

This project is a prototype developed during a hackathon. While it demonstrates core concepts around privacy, safety, and documentation, it is not a production-ready tool and should not be relied upon as a substitute for professional or emergency services.
